1. Contexto y objetivo Crea la pantalla principal de una PWA de gestión de suscripciones personales. El usuario quiere ver de un vistazo: Cuánto tiene comprometido al mes / año. Qué viene pronto. Qué suscripciones son más “problemáticas”. El estilo visual debe ser neobrutalismo utilitario: bordes negros gruesos, bloques bien definidos, sin tarjetas genéricas con sombra suave ni gradiente azul‑morado. 2. Estilo global Tipografía Heading / UI general: Space Grotesk (sans geométrica). Usar pesos 500–700 para títulos, 400 para labels y texto corto. Números y datos (montos, fechas, IDs): Fira Code (monoespaciada). Montos y cifras importantes siempre en Fira Code para reforzar la sensación de “dato contable”. Paleta de colores Fondo base app: #F4F1EC (gris cálido muy claro). Texto principal: #18181B. Acento principal / CTA: #FF5A1F (naranja intenso). Estado “alerta/zombie”: #D7263D (rojo ladrillo). Estado “ok/activa”: #1E8E4D (verde mate). Fondos de bloques secundarios: #FFFFFF y un gris muy claro #F3F4F6 para alternar filas. Tratamiento visual (neobrutalismo) Bordes: La mayoría de los bloques (cards, filas, badges) con borde negro de 2–3 px. Sombras: Si se usan, que sean sombras duras sin blur, desplazadas 2–4 px, no sombras suaves difusas. Nada de gradientes azules/morados genéricos ni “cards” de UI kits estándar. 3. con el H1. Contenido: H1 (Space Grotesk): “Suscripciones” (o “Sin Sorpresas Mensuales”). Subtítulo pequeño (párrafo o small): “Resumen de [mes] [año]”. Estilo Sin caja fancy: puede ser simplemente texto en la parte superior, con un borde inferior negro de 2 px para separarlo del contenido. 4.2. Bloque “Resumen de compromiso” Semántica:
. Contenido: H2 (id="resumen-heading"): “Resumen de compromiso”. Texto en Fira Code grande: “Comprometido este mes” + monto (p.ej. $120.000). Texto más pequeño con total anual (equivalente): “Equivalente anual: $X”. Pequeño desglose por persona (puede ser 2–3 etiquetas): “Yo: $XX.XXX”, “Pareja: $YY.YYY”, “Familia: $ZZ.ZZZ”. Estilo visual Bloque rectangular de ancho completo (o columna en desktop). Fondo naranja claro (puedes derivarlo de #FF5A1F bajando saturación), texto oscuro. Borde negro de 3 px, sin sombra o con sombra dura mínima. Montos en Fira Code grande, centrados o alineados a la izquierda con amplio espacio. 4.3. Bloque “Alertas y zombies” Semántica:
. Contenido: H2: “Alertas”. Dentro, 2–3 “chips”/cards horizontales: Trials: “Pruebas que terminan pronto: N”. Zombies: “Suscripciones problemáticas: M”. Opcional: “Revisar más tarde”, etc. Estilo Cada “chip” es una mini card: fondo blanco, borde negro 2 px, ligera sombra dura opcional. Texto en Space Grotesk; cifra principal en Fira Code. Zombie: usar color de acento rojo ladrillo para icono, borde o detalle del chip. 4.4. Bloque “Próximos cobros” Semántica:
